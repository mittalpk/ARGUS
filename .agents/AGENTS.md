# Project Rules

- **Human-like Code & Documentation**: All files written in this repository must look like they were written by an experienced human engineer. Avoid overly templated, repetitive, or obviously AI-generated boilerplate.
- **Production & Public-Ready**: All code, documentation, comments, and commit messages must be clean, professional, well-structured, and ready for a public GitHub repository. No placeholders, temporary comments (unless standard TODOs), or low-effort outputs.
